# Phase 08 — 综合 → 给项目负责人的产品决策输入

## 模型选择
- **opus 全程**(综合 + 立场 + 决策辅助,是最重要的判断任务)

## 目标
**这不是产品方案**。这是给一个早期产品负责人的**决策辅助材料**。读完后他/她要能:
- 对市场有质的认知提升
- 对两条产品路径(私密 vs 展示)有明确推荐 + 理由(不骑墙)
- 知道接下来 4 周该做什么
- 看到诚实的调研局限性

## 调研内容(综合,不再有新搜索)
读 `analysis/01_*.md` ~ `analysis/07_*.md` + `citations/citations_v2.json` 中所有引用 + `raw_data/phase_0X/` 的 TL;DR(必要时打开关键原文)。

### 必须回答的 6 个问题

1. **我现在了解的市场全景** — 三个目标人群、生态、竞争格局
2. **两条路径权衡 + 我的推荐 + 理由** — 默认私密 vs 默认展示 vs 混合
3. **真实约束清单** — 技术、合规、商业
4. **用户意愿的真实证据** — 综合 Phase 06,给出最 sturdy 的结论
5. **给项目负责人的下一步建议** — 4 周内该读、该聊、该想清楚的事
6. **调研局限性** — 诚实

## 输出要求

### 结构
保存到 `v2_final_report.md`,章节:

1. **TL;DR(< 800 字)** — 让没时间的人读这一段就能做决策
2. **我现在了解的市场全景**
   - 三个目标人群(Marketing / Design / Consulting)真实日常 + TAM
   - Newsletter / 职业身份产品生态全景
   - 主要竞争对手(显性 + 隐性)
3. **两条路径深度权衡**
   - 默认私密:利、弊、与"反哺 B 端"契合度
   - 默认展示:利、弊、同上
   - 混合模式:具体设计
   - **我的明确推荐 + 4-6 条具体理由 + 我能预见的最强反对意见 + 我的回应**
4. **真实约束清单**
   - 技术:API、数据获取、AI 真实能力
   - 合规:GDPR、CCPA、EU AI Act
   - 商业:B 端兑现路径 + 用户感知红线
5. **用户意愿的真实证据**
   - 付费意愿区间(基于真数据)
   - 数据贡献意愿模型
   - 公开意愿强度分布
6. **给项目负责人的下一步建议**(4 周可执行清单)
   - 该读:5 本书 / 文章 / report 名单
   - 该聊:5 类人(创始人 + KOL + 法务 + B 端客户 + 早期用户)
   - 该想清楚:3-5 个最优先未解决的问题
7. **调研局限性**
   - 数据可访问性 limitations
   - 样本偏差
   - 未访谈真实用户(纯桌面调研)
   - 其他

### 质量底线
- **不骑墙**:必须给出明确推荐路径。可以有 trade-off,但不能"两个都好"
- **不堆砌**:不要把 Phase 01-07 的内容拼起来。必须有跨 Phase 的洞察
- **承认不知道**:Phase 中证据弱的结论,在 final report 中要明说"证据不足"
- **可执行**:next steps 不能是空话(不能写"do user research"),要写"和 X 类型的 3 个人聊"
- **简洁**:目标是项目负责人能 30 分钟读完(8-12 页正文)

### 引用
final report 中关键结论必须有 [VXXX] 引用 ID,且与 citations_v2.json 一致。

## 写作风格
- 直接、不客套
- 用具体例子和真实数字让结论有质感
- 不要"建议进一步研究"这种 cop-out
- 关键判断后有"为什么我相信这个"段
- 不写 emoji,不写 marketing fluff

## 期望页面规模
8-15 页(6000-12000 字)。 偏向简洁。
