# Strava — Public Layer

## Source date range: 2024–2025

## Scale
- 180M registered users (2025, up 35M YoY)
- 135M users in 190+ countries (2024)
- 4B activities published on Strava in 2025
- 14B kudos given in 2025 (up 2B YoY)

## Revenue
- 2023: $132.3M
- 2024: $163.4M (reported); some sources cite $338M–$415M ARR (Latka higher estimate)
- 2025: $415M–$500M ARR (various estimates)
- 90% revenue from premium subscriptions ($79.99/year or $11.99/month)
- Valuation: $2.2B (latest round)

## Default Public Setting
- Official source: support.strava.com (Privacy Defaults article, updated March 2026)
- New accounts: Profile = Everyone; Activities = Everyone; Maps = Everyone
- First activity uploaded → first/last 200m of future routes auto-hidden (privacy zone)
- The "70% public" claim: no primary source found; however, since default = public and most users don't change settings, majority-public is reasonable inference
- Treat as: "default public → inferred majority public; no verified 70% stat"

## Activity Feed (Public Layer)
- Activity feed: chronological + segment leaderboards (public by default)
- Segments: public KOM/QOM records create competitive public benchmarks
- Clubs: semi-public (joinable communities)
- Challenges: sponsored by brands; public participation

## B2B Data Layer
- Strava Metro: free to 3,500+ qualified city planners, governments, researchers
- Aggregated, anonymized route/activity data for infrastructure planning
- Revenue contribution: small (Metro primarily used as PR/goodwill; not primary revenue)
- Key: only location/activity patterns shared with B2B — health metrics (HR, power) NOT sold

## D30 / D90 Retention
- Not publicly disclosed
- High retention indicators: 4B activities/year / 180M users = ~22 activities/user/year
- Premium retention: annual subscription creates 12-month floor; renewal rates not disclosed

## Design Pattern We Could Copy
- Default public activity feed → social accountability loop → premium conversion
- Heart rate / health data: private layer creates trust
- Metro: aggregate public data → B2B intelligence without touching sensitive private layer

## Fit with "Feed B-Side Data" Goal
- HIGH fit model: public signals (activity, engagement) → B2B intelligence; private health data = trust anchor
- This is the hybrid model most relevant to our product
