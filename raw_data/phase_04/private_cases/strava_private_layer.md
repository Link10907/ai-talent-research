# Strava — Private Layer

## Source date range: 2024–2025

## The Private Layer Within a Default-Public Platform
Strava's default is public (see public_cases/strava_public_layer.md), but it has meaningful private data layers:
- Heart rate data: visible only to user by default
- Medical conditions / injury notes: private
- Privacy zones: first/last 200m of activity hidden by default after first upload
- Gear data, personal records: user-controlled
- Beacon (real-time location sharing): opt-in, limited recipients

## Default Privacy Settings (Official Strava Source)
When creating a new account:
- Profile: Everyone (public)
- Activities: Everyone (public)
- Activity maps: Everyone (public)
After first activity upload → start/end location auto-hidden within 200m

Source: support.strava.com/hc/en-us/articles/360034758331 (official Strava support page, updated March 2026)

## The "70% Public" Claim
- Widely cited but primary source not found in official Strava documents
- Strava support page confirms default = public for all new accounts
- The 70%+ public stat likely derives from: since default = public AND most users never change defaults, majority remain public
- No official Strava press release or annual report with this exact figure found
- Verdict: treat as "default = public → inferred majority public" not a measured statistic

## Data Sold / Licensed
- Strava Metro: aggregated, anonymized activity data licensed to 3,500+ city governments and urban planners
- Free for qualifying government/advocacy orgs; commercial tiers exist
- Insight: the "private" heart rate and health metrics are NOT sold; only route/location patterns

## Fit with "Feed B-Side Data" Goal
- MODERATE fit: route/activity data B2B works; health biomarkers remain locked
- User trust maintained by keeping sensitive health data off B2B channel
