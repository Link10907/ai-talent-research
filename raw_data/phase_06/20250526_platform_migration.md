# TLDR: Platform Migration Patterns — What the Data Shows

**Date collected**: 2025-05-26
**Source type**: TechCrunch, DemandSage, BluePilot, Backlinko, Social Shepherd

## Migration Events & Outcomes
| Event | Migrators | 6-Month Retention |
|-------|---------|-----------------|
| Twitter → Mastodon (Nov 2022) | 1.8M MAU peak | ~10% of registered accounts |
| Twitter → Threads (July 2023) | 41M DAU day 1 | 25% MAU retained (18 months) |
| Twitter → Bluesky (Nov 2024) | +13M in 6 weeks | −64% engagement by June 2025 |
| Read.cv shutdown (May 2025) | Unknown total | Dispersed, no successor |
| Polywork shutdown (Jan 2025) | Unknown total | Dispersed, no successor |

## The Pattern
1. Trigger event (policy change, acquisition, shutdown) → spike
2. Initial migration is high but shallow (curiosity-driven)
3. Retention requires: network effect at destination + new habit formation
4. Without clear successor network, users revert to incumbent or disperse

## The LinkedIn Exception
LinkedIn has survived every "LinkedIn killer" because:
- Recruiter network effect is irreplaceable without matching recruiter buy-in
- Data portability is low (connections don't move)
- Job-market loop creates hard dependency

## What Triggers Real Migration
- Platform CLOSES (not just declines) — forced migration
- Incumbent becomes hostile (censorship, blocking, price shock)
- New platform has instant social graph (Threads via Instagram = 175M in one month)

## Confidence
High for public platforms (all figures from press). Unknown for Read.cv/Polywork (no user data released).
