# "Hire Me" Tags — Usage Across Platforms

## GitHub Bio / Profile
- GitHub has 100M+ registered developers (2023 milestone)
- No official count of "hire me" / "open to work" profile signals
- GitHub profile README "for hire" badges are common in open-source community
- GitHub itself launched "Available for hire" status option in profile settings (exact date unclear)
- **Estimate**: Low single-digit % of active GitHub users signal availability publicly

## X / Twitter Bio "Open to Opportunities"
- No platform-level data released
- Anecdotal: common in tech/design/PM communities on X
- Search for "open to work" on X returns millions of results — but no count
- **Estimate**: ~1–3% of tech/creator professional X accounts signal availability (inferred)

## Personal Sites / Portfolio Pages
- Widely used: "Available for freelance/contract/full-time" on personal sites
- Carrd, Webflow, personal domains commonly include hire-me CTAs
- No aggregate data available

## Summary Table
| Platform | Signal Type | Rough % with Signal | Data Quality |
|----------|------------|---------------------|-------------|
| LinkedIn (public badge) | #OpenToWork frame | ~3–4% of members | High |
| LinkedIn (private) | Backend recruiter signal | ~22% of members | High |
| GitHub | "Available for hire" | ~2–5% of active devs | Low (estimated) |
| X/Twitter | Bio text signal | ~1–3% of tech accounts | Very Low (estimated) |
| Wellfound | Candidate status | Near 100% (platform purpose) | High |

## Key Insight
LinkedIn remains the dominant structured signal platform. Wellfound is purpose-built for this (all candidates explicitly available). GitHub "for hire" is informal but reaches developers directly. No cross-platform aggregation exists.

## Evidence Quality
- LinkedIn: **High** (see linkedin_open_to_work.md)
- GitHub/X: **Low** (no published data; estimates only)
- Wellfound: **High** (platform purpose = job seeking)
