First-principles Product Thesis
第一性目的

用真实 C 端价值，补全 Metix.ai 的候选人数据生态。

项目的核心目的，是持续获取候选人的能力、技能、简历反馈、行动证明和机会偏好，反哺 Metix.ai / openjobs-recruiter 的候选人搜索、匹配和触达。前台必须先给用户明确收益，后台才能形成高质量职业数据。

用户为什么来AI 变化太快，需要知道新方向、新技能、新机会。
为什么现在成立AI 降低学习和跨领域试错门槛，职业路径被重新打开。
用户得到什么职业雷达、技能树、证明框架、全球机会可见性。
平台沉淀什么兴趣、技能、项目、Proof、简历反馈和机会偏好。

C 端吸引力：把焦虑转成下一步职业路径

用户不是来完成平台布置的任务，而是来获得一条更清楚的行动路径：我关注的方向发生了什么，我应该补什么技能，我可以用什么真实经历证明自己。

User-facing story

我不一定马上找工作，但我想知道 AI 时代我还能往哪里走。

用户先选择一个感兴趣方向。系统把一手变化和真实岗位需求翻译成职业含义，再给出可选的 Proof Prompt。用户用自己的真实工作、已有项目、学习实验或 workflow 来填充，而不是从零做一个平台规定的大项目。

1看到一条一手变化或岗位需求信号。
2系统解释它影响哪些岗位与技能。
3用户选择一个 Proof Prompt，用自己的经历填充。
4形成可私密保存、分享或授权给机会方查看的职业证明。
更少信息噪声把一手变化翻译成“对我的职业有什么影响”。
更清楚该学什么技能树由趋势、岗位需求和证明方式共同校准。
更容易整理自己用户把已有经历、工作流和想法实验转成 proof，而不是重新包装简历。
更安全地被看见默认私密。只有用户准备好并授权时，才连接到全球机会。

为什么这样设计

前台解决用户的 AI 时代职业焦虑，后台沉淀可用于 B 端招聘生态的数据。

01 变化太快用户需要一手变化和真实岗位需求。
02 门槛降低AI 让跨领域学习和试错更容易。
03 选择更重要想做什么、判断什么、如何行动更能区分人。
04 证明要升级技能不能只靠声明，需要行动过程、人的判断和结果信号。
用户价值是入口，能力数据是资产，B 端生态是最终目的。

模块设计

每个模块都先服务用户价值，再沉淀可用于 B 端招聘匹配的职业数据。

1

Career Radar
职业雷达

把 AI 时代一手变化翻译成职业含义。

产品设计

用户选择关注领域，例如 AI Marketing、AI Design、AI Ops。系统聚合官方发布、论文、开源项目、工具更新、岗位变化和 Metix.ai B 端需求信号。

用户价值

用户知道“这件事对我意味着什么，我应该关注什么技能”。

数据沉淀

形成 Interest Graph 和 Attention Signal。

2

Skill Tree
技能树

从岗位需求和趋势反推技能路径。

产品设计

每个领域有动态技能树，包含基础能力、AI 工具、工作流能力、行业知识和证明方式。

用户价值

用户知道该学什么、先学什么、如何用已有经历证明。

数据沉淀

形成 Skill Graph 和 Learning Intent。

3

Action Proof
行动证明

把真实经历、想法和 workflow 沉淀为可信职业资产。

产品设计

系统提供 Proof Prompt。用户填写背景、行动、工具、人的判断、产出、来源证据和可见性范围。

用户价值

用户证明“我如何行动”，而不是只声明“我会什么”。

数据沉淀

形成 Evidence Graph，可用于用户授权后的机会匹配。

4

Global Visibility
全球可见性

把行动证明连接到真实招聘机会。

产品设计

用户默认私密积累。打开可见性后，系统在用户授权范围内把技能树和行动证明用于具体机会匹配。

用户价值

用户在准备好后，让全球机会看到自己的方向、技能和行动力。

数据沉淀

形成 Opportunity Graph。

证明如何实现：从 Proof Prompt 到 Evidence Ledger

Proof Prompt 不是平台任务，而是可选择的证明框架。可信度来自来源、时间戳、证据完整度和用户授权，而不是 AI 文案本身。

Radar 事件一手变化或岗位需求
技能映射关联技能树节点
Proof Prompt生成证明模板
用户填充真实经历 / 项目 / workflow
可信展示私密、分享或机会可见

示例：AI Marketing / Lead Enrichment

Proof Prompt

你可以证明的技能：Lead Enrichment、GTM Automation、Prompt Ops

推荐证明方式：用一个脱敏线索样本，展示你如何用 AI 工具完成线索补全、优先级判断和个性化触达准备。

  • 输入：10–20 条脱敏公司或客户线索。
  • 行动：说明工具链，例如 ChatGPT / Claude / Clay / Sheets / CRM。
  • 人的判断：说明哪些字段由 AI 生成，哪些判断由你确认或修正。
  • 产出：展示脱敏 workflow、样例输出或结果摘要。
  • 可见性：默认私密；用户可选择分享链接或开放给匹配机会。

技能点亮不是“自称掌握”

  • Interested：用户关注技能。
  • Learning：用户收藏资源或阅读 Radar。
  • Practiced：用户提交练习或已有 workflow。
  • Proved：用户提交 Action Proof。
  • Verified / Endorsed：证据有外部来源、第三方背书或 B 端反馈。

可信度等级

  • L0 Self-claimed：用户自称。
  • L1 Structured：按模板结构化说明。
  • L2 Artifact-linked：附链接、截图、文件、repo。
  • L3 Source-verified：OAuth / API / metadata 可确认。
  • L4 Outcome signal：招聘方收藏、面试、客户采用或结果指标。

Metix.ai 的优势：C 端不是孤立产品，而是接入已有 B 端需求

OpenJobs Recruiter 已经覆盖招聘任务、JD 结构化、候选人搜索、触达、候选人落地页、约面、权益和订阅支付。C 端产品可以把“技能树与行动证明”接入这个招聘闭环。

已有 B 端能力

  • 招聘任务与岗位上下文管理
  • JD 结构化与候选人深度搜索
  • 多维度匹配评分
  • 邮件触达与多轮跟进
  • 候选人落地页、短链与简历回流
  • 面试预约、权益解锁、订阅支付

C 端数据如何进入闭环

用户资产B 端用途
兴趣领域理解候选人未来方向
技能树提升岗位匹配精度
行动证明辅助 recruiter 判断真实能力
机会偏好减少无效触达，提高回应概率
