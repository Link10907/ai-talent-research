First-principles Product Thesis
第一性目的

用真实 C 端价值，补全 Metix.ai 的候选人数据生态。

项目的核心目的，是持续获取候选人的能力、技能、简历反馈、行动证明和机会偏好，反哺 Metix.ai / openjobs-recruiter 的候选人搜索、匹配和触达。前台必须先给用户明确收益，后台才能形成高质量职业数据。

用户为什么来AI 变化太快，需要知道新方向、新技能、新机会。
为什么现在成立AI 降低学习和跨领域试错门槛，职业路径被重新打开。
用户得到什么职业雷达、账号连接、能力图谱、证明框架、成长建议。
平台沉淀什么兴趣、现有能力、潜在能力、泛化能力、Proof、简历反馈和机会偏好。

C 端吸引力：把已有资产整理成下一步成长路径

用户不是从零填写资料，也不是完成平台布置的任务。用户把 GitHub、作品、文章、项目、简历等已有职业资产接进来，系统帮他看清自己会什么、还能往哪里走、还缺什么证明。

用户故事

我不一定马上找工作，但我想知道 AI 时代我还能往哪里走。我希望系统先理解我已经做过什么，再告诉我哪些技能值得补、哪些能力可以迁移、哪些 proof 可以让我被机会看到。

1连接账号或提交作品链接，形成初始职业资产。
2系统识别现有能力、潜在能力和可泛化能力。
3结合职业雷达和 B 端岗位需求，推荐技能路径。
4用户补充 proof，私密保存或授权展示。
更清楚自己会什么

把分散在代码、作品、文章、项目和简历里的能力整理成一张图谱。

更知道自己能转向哪里

识别潜在能力和可迁移能力，而不是只按过去职位定义用户。

更容易持续成长

围绕关注领域、技能缺口和 Proof 缺口做轻量更新。

更安全地被看见

默认私密。用户准备好并授权后，才连接全球机会。

账号连接与证据源：先把用户已有资产接进来

第一步不是让用户填表，而是把能证明能力的账号、作品和项目源接入。系统从中抽取技能、工具、产出、角色、时间线和证据强度。

GitHub

repo、commit、PR、issue、release、README、language，用于工程、自动化、AI 工具类 proof。

Portfolio / Design

个人站、Behance、Dribbble、Figma 链接，用于设计、创意和产品表达 proof。

Writing / Research

Substack、Medium、博客、论文、技术文章，用于观点、研究和结构化表达 proof。

Project / Resume

Notion、Docs、Slides、脱敏截图、简历和 LinkedIn export，用于基础档案和工作流 proof。

连接账号不是为了抓取一切，而是让用户把“我做过什么”转化为“我能证明什么”。

全面能力图谱：现有能力、潜在能力、可泛化能力

AI 时代的候选人评价不能只看过去职位。系统需要从多源证据中判断用户现在能做什么、可能适合什么、哪些能力可以迁移到新领域。

现有能力

用户已经有明确证据支持的技能，可作为 B 端高置信匹配字段。

  • GitHub / 作品 / 文章 / 项目 / 简历经历
  • 状态：Self-claimed 到 Source-verified
  • 用于候选人搜索和匹配

潜在能力

用户可能具备基础但证据不足的能力，主要用于成长推荐。

  • 相邻技能、关注领域、学习记录
  • 状态：Interested / Learning / inferred
  • 不直接作为强招聘证明

可泛化能力

跨岗位、跨行业迁移的能力模式。

  • 结构化表达、问题拆解、自动化思维
  • 数据判断、系统设计、产品化能力
  • 用于转型建议和非传统背景发现

模块设计

每个模块都先服务用户价值，再沉淀可用于 B 端招聘匹配的职业数据。

1

Career Radar

把一手变化翻译成职业含义。

产品设计

聚合官方发布、论文、开源项目、工具更新、岗位变化和 Metix.ai B 端需求信号。

用户价值

知道“这件事对我意味着什么，我应该关注什么技能”。

2

Capability Graph

把多源资产转成三层能力图谱。

产品设计

系统把连接账号、作品、简历和项目文件映射到现有、潜在、泛化能力。

用户价值

知道自己已经证明了什么，还可以向哪里成长。

3

Action Proof

把真实经历和 workflow 沉淀为可信职业资产。

产品设计

系统提供 Proof Prompt，用户填写背景、行动、工具、人的判断、产出、来源证据和可见性。

用户价值

证明“我如何行动”，而不是只声明“我会什么”。

4

Growth Check-in

把职业资产变成持续更新系统。

产品设计

每周推送职业雷达、技能缺口、知识推荐和可补充 proof。

用户价值

不只在求职时更新资料，而是持续更新职业资产。

证明如何实现：从账号连接到 Evidence Ledger

Proof Prompt 不是平台任务，而是可选择的证明框架。可信度来自来源、时间戳、证据完整度和用户授权，而不是 AI 文案本身。

连接证据源GitHub / 作品 / 文章 / 项目 / 简历
抽取能力线索技能、工具、角色、产出、时间线
映射能力图谱现有 / 潜在 / 泛化能力
生成 Proof Prompt提示用户补齐证据缺口
可信展示私密、分享或机会可见

示例：GitHub 连接后的能力证明 Engineering Proof

系统读取：用户授权的 repo、commit、release、README、语言、issue / PR 参与记录。

系统推断：现有能力 = Python / API integration / workflow automation；潜在能力 = AI agent orchestration；泛化能力 = 系统拆解、调试、持续迭代。

Proof Prompt：选择一个 repo 或 PR，说明问题、你的贡献、工具链、关键判断和结果。系统生成 proof card，但由用户确认后才展示。

技能点亮规则

  • Interested：关注技能。
  • Learning：收藏资源或阅读 Radar。
  • Practiced：提交练习或已有 workflow。
  • Proved：提交 Action Proof。
  • Verified / Endorsed：外部来源、第三方背书或 B 端反馈。

可信度等级

  • L0 Self-claimed：用户自称。
  • L1 Structured：按模板说明。
  • L2 Artifact-linked：附链接、截图、文件、repo。
  • L3 Source-verified：OAuth / API / metadata 可确认。
  • L4 Outcome signal：招聘方收藏、面试、客户采用或结果指标。

成长打卡体系：让用户持续回来，也让数据持续变新

打卡不是签到，而是围绕“我关注什么、我学了什么、我做了什么、我能证明什么”更新职业资产。

每周用户看到什么

  • 本周关注领域变化。
  • 哪些技能与用户现有能力最接近。
  • 哪些技能正在被 B 端岗位需求提到。
  • 本周可以补充哪一条 proof。
  • 能力图谱本周有什么变化。

平台获得什么

  • Freshness Signal：用户是否仍在学习和行动。
  • Learning Intent：用户正在向哪里成长。
  • Evidence Growth：候选人数据持续补证据。
  • Opportunity Fit：更准确判断适合机会和触达意愿。
简历是低频更新；成长打卡让职业资产变成持续更新的系统。

Metix.ai 的优势：C 端不是孤立产品，而是接入已有 B 端需求

OpenJobs Recruiter 已经覆盖招聘任务、JD 结构化、候选人搜索、触达、候选人落地页、约面、权益和订阅支付。C 端产品可以把能力图谱与行动证明接入这个招聘闭环。

已有 B 端能力

  • 招聘任务与岗位上下文管理
  • JD 结构化与候选人深度搜索
  • 多维度匹配评分
  • 邮件触达与多轮跟进
  • 候选人落地页、短链与简历回流
  • 面试预约、权益解锁、订阅支付

C 端数据如何进入闭环

用户资产B 端用途
现有能力提高候选人搜索和匹配精度
潜在能力发现可培养、可转型人才
可泛化能力支持跨行业、跨岗位推荐
行动证明辅助 recruiter 判断真实能力
机会偏好减少无效触达，提高回应概率
