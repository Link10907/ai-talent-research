First-principles Product Thesis
第一性目的

用真实 C 端价值，补全 Metix.ai 的候选人数据生态。

项目的核心目的，是持续获取候选人的能力、技能、简历反馈、行动证明和机会偏好，反哺 Metix.ai / openjobs-recruiter 的候选人搜索、匹配和触达。前台必须先给用户明确收益，后台才能形成高质量职业数据。

用户为什么来AI 变化太快，需要知道新方向、新技能、新机会。
为什么现在成立AI 降低学习和跨领域试错门槛，职业路径被重新打开。
用户得到什么职业雷达、技能树、证明框架、全球机会可见性；账号连接和成长打卡是增强层。
平台沉淀什么兴趣、技能、项目、Proof、简历反馈、机会偏好，以及持续更新的能力图谱。

C 端吸引力：把焦虑转成下一步职业路径

用户不是来完成平台布置的任务，而是来获得一条更清楚的行动路径：我关注的方向发生了什么，我应该补什么技能，我可以用什么真实经历证明自己。

User-facing story

我不一定马上找工作，但我想知道 AI 时代我还能往哪里走。

用户先选择一个感兴趣方向。系统把一手变化和真实岗位需求翻译成职业含义，再给出可选的 Proof Prompt。账号连接只是帮助系统理解用户已有资产，不替代职业雷达、技能树和行动证明这条主线。

1看到一条一手变化或岗位需求信号。
2系统解释它影响哪些岗位与技能。
3用户结合已有资产选择一个 Proof Prompt。
4形成可私密保存、分享或授权给机会方查看的职业证明。
更少信息噪声把一手变化翻译成“对我的职业有什么影响”。
更清楚该学什么技能树由趋势、岗位需求和证明方式共同校准。
更容易整理自己用户把已有经历、作品、代码、文章、工作流和想法实验转成 proof。
更安全地被看见默认私密。只有用户准备好并授权时，才连接到全球机会。

为什么这样设计

前台解决用户的 AI 时代职业焦虑，后台沉淀可用于 B 端招聘生态的数据。

01 变化太快用户需要一手变化和真实岗位需求。
02 门槛降低AI 让跨领域学习和试错更容易。
03 选择更重要想做什么、判断什么、如何行动更能区分人。
04 证明要升级技能不能只靠声明，需要行动过程、人的判断和结果信号。
用户价值是入口，能力数据是资产，B 端生态是最终目的。

账号连接与证据源：作为 Action Proof 的输入层

这一层是对原方案的补强，不是替代。用户仍然通过职业雷达、技能树和 Proof Prompt 形成主路径；账号连接的作用，是把用户已经做过的东西带进系统，减少手动填写，提高 proof 可信度。

GitHub

repo、commit、PR、issue、release、README、language，用于工程、自动化、AI 工具类 proof。

Portfolio / Design

个人站、Behance、Dribbble、Figma 链接，用于设计、创意和产品表达 proof。

Writing / Research

Substack、Medium、博客、论文、技术文章，用于观点、研究和结构化表达 proof。

Project / Workflow

Notion、Docs、Slides、脱敏截图、workflow diagram，用于业务流程和 AI workflow proof。

Resume / Profile

简历、LinkedIn export、手动经历，用于基础候选人档案和履历反馈。

连接账号不是为了抓取一切，而是让用户把“我做过什么”转化为“我能证明什么”。

全面能力图谱：接入 Skill Tree，而不是替代 Skill Tree

原有技能树回答“一个方向需要什么能力”；能力图谱回答“这个用户现在有什么能力、可能具备什么能力、哪些能力可以迁移”。两者结合后，产品更适合做成长推荐和 B 端候选人理解。

现有能力

用户已有明确证据支持的技能，可作为 B 端匹配中的高置信字段。

  • 来源：GitHub、作品链接、文章、项目文件、简历经历。
  • 状态：Self-claimed / Structured / Artifact-linked / Source-verified。

潜在能力

用户可能具备基础但证据不足的能力，主要用于成长推荐。

  • 来源：相邻技能、关注领域、学习记录、类似项目经验。
  • 不直接作为强招聘证明。

可泛化能力

跨岗位、跨行业迁移的能力模式。

  • 例如结构化表达、问题拆解、自动化思维、系统设计、数据判断。
  • 用于转型建议和非传统背景人才发现。

模块设计

恢复原主线：职业雷达、技能树、行动证明、全球可见性。账号连接、能力图谱和成长打卡分别加到对应模块中作为增强机制。

1

Career Radar
职业雷达

把 AI 时代一手变化翻译成职业含义。

产品设计

用户选择关注领域，例如 AI Marketing、AI Design、AI Ops。系统聚合官方发布、论文、开源项目、工具更新、岗位变化和 Metix.ai B 端需求信号。

新增增强

Radar 会结合用户已连接资产，解释“这个变化与你已有能力有什么关系”。

数据沉淀

形成 Interest Graph 和 Attention Signal。

2

Skill Tree
技能树

从岗位需求和趋势反推技能路径。

产品设计

每个领域有动态技能树，包含基础能力、AI 工具、工作流能力、行业知识和证明方式。

新增增强

技能树叠加用户能力图谱，区分现有能力、潜在能力和可泛化能力，帮助用户找到相邻成长路径。

数据沉淀

形成 Skill Graph、Learning Intent 和 Capability Graph。

3

Action Proof
行动证明

把真实经历、想法和 workflow 沉淀为可信职业资产。

产品设计

系统提供 Proof Prompt。用户填写背景、行动、工具、人的判断、产出、来源证据和可见性范围。

新增增强

账号连接提供证据来源，减少用户从零填写，并提升 evidence confidence。

数据沉淀

形成 Evidence Graph，可用于用户授权后的机会匹配。

4

Global Visibility
全球可见性

把行动证明连接到真实招聘机会。

产品设计

用户默认私密积累。打开可见性后，系统在用户授权范围内把技能树和行动证明用于具体机会匹配。

新增增强

成长打卡让职业资产持续变新，帮助 B 端看到用户最近是否仍在学习、行动和更新。

数据沉淀

形成 Opportunity Graph、Freshness Signal 和 Proof Growth。

证明如何实现：从 Proof Prompt 到 Evidence Ledger

Proof Prompt 不是平台任务，而是可选择的证明框架。可信度来自来源、时间戳、证据完整度和用户授权，而不是 AI 文案本身。

Radar 事件一手变化或岗位需求
技能映射关联技能树节点
账号 / 作品证据GitHub、作品、文章、简历
Proof Prompt提示用户补齐证据缺口
可信展示私密、分享或机会可见

示例：GitHub 连接后的工程类证明

Engineering Proof

系统读取：用户授权的 repo、commit、release、README、语言、issue / PR 参与记录。

系统推断：现有能力 = Python / API integration / workflow automation；潜在能力 = AI agent orchestration；泛化能力 = 系统拆解、调试、持续迭代。

Proof Prompt：选择一个 repo 或 PR，说明问题、你的贡献、工具链、关键判断和结果。系统生成 proof card，但由用户确认后才展示。

技能点亮规则

  • Interested：关注技能。
  • Learning：收藏资源或阅读 Radar。
  • Practiced：提交练习或已有 workflow。
  • Proved：提交 Action Proof。
  • Verified / Endorsed：外部来源、第三方背书或 B 端反馈。

可信度等级

  • L0 Self-claimed：用户自称。
  • L1 Structured：按模板说明。
  • L2 Artifact-linked：附链接、截图、文件、repo。
  • L3 Source-verified：OAuth / API / metadata 可确认。
  • L4 Outcome signal：招聘方收藏、面试、客户采用或结果指标。

成长打卡：作为留存机制，补充到原主线之后

打卡不是签到，也不是新主产品。它是职业雷达、技能树和行动证明的持续更新机制，让用户每周补一点证据、更新一点方向。

每周用户看到什么

1
关注领域变化

职业雷达总结与用户相关的一手变化。

2
技能树变化

哪些技能变热，哪些技能与已有能力相邻。

3
知识推荐

根据潜在能力和证明缺口推荐文章、案例、项目范式。

4
Proof 缺口

提醒用户补充一个链接、截图、项目说明或结果信号。

平台获得什么

A
Freshness Signal

知道用户最近是否仍在学习、行动和更新。

B
Learning Intent

知道用户正在向哪些方向迁移。

C
Evidence Growth

候选人数据从静态简历变成持续更新的证据账本。

D
Opportunity Fit

更准确判断用户适合什么机会、愿不愿意被触达。

简历是低频更新；成长打卡让职业资产变成持续更新的系统。

Metix.ai 的优势：C 端不是孤立产品，而是接入已有 B 端需求

OpenJobs Recruiter 已经覆盖招聘任务、JD 结构化、候选人搜索、触达、候选人落地页、约面、权益和订阅支付。C 端产品可以把技能树与行动证明接入这个招聘闭环。

已有 B 端能力

  • 招聘任务与岗位上下文管理
  • JD 结构化与候选人深度搜索
  • 多维度匹配评分
  • 邮件触达与多轮跟进
  • 候选人落地页、短链与简历回流
  • 面试预约、权益解锁、订阅支付

C 端数据如何进入闭环

用户资产B 端用途
现有能力提高候选人搜索和匹配精度
潜在能力发现可培养、可转型人才
可泛化能力支持跨行业、跨岗位推荐
行动证明辅助 recruiter 判断真实能力
机会偏好减少无效触达，提高回应概率
